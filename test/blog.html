Stop Abusing the JavaScript Ternary Operator

Why do programmers give variables descriptive names? Why do they decompose large problems into smaller functions? Why do they split up functionality into separate modules, each with their own purpose? Why do they create abstractions? And why, oh why, do they add comments to their code?

One primary answer, as I'm sure you know, is to make code easy to understand--to make it clear. The point of using descriptive names is to make it easy to identify what code is doing. The point of decomposition is to make it easy to understand simple parts without getting overwhelmed with the entire problem. The point of comments is to add further explanation--further clarity--when code itself doesn't suffice.

In other words, the point of many common coding practices is to make your code clear and comprehendible.
